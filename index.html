Botter Authoring Portal

Browser-based editors for biomes, items, affixes, and tiles. Author content locally, export JSON, share back via GitHub issues / Discord.

Editors

live Tile Atlas Viewer

Browse all 6,945 DCSS tiles by category, biome, class, or subdir. Filter, search, click to copy paths. Read-only — pairs with the editors below.

 live Biome Editor

Visual editor for floor, wall, edge-overlay, and sigil sets per biome. Replace tiles by picker. Exports a complete biomes.json.

 live Item Editor

Slot-tabbed editor (1H sword / helm / armor / shield / boots / ring / amulet). Edit stats, drop weights, flavor tags, lore. Exports per-slot JSON slices.

 live Affix / Enchant Editor

Author new affixes (rarity-tiered stat bonuses) with a color picker, stat dropdown, and live preview card. Exports a complete affixes.json.

 live Drop Tuning

Centralized sliders for every drop-rate dial: meta-rarity, quality tail bias, enchant chance, enchant-combo chance, recolor odds. Exports drop_tuning.json.

 live Item Roll Flowchart

Visual reference: every way an item can roll, from Common junk to Sublime Primal Prismatic. Tooltip cards mirror the in-game widget so you can read the same fields the player sees.

 live Build Matrix

15 species × 10 spell archetypes grid. Cells coloured VIABLE / WEAK / UNBUILT / NO ANCHOR. Click a cell to see the item pool, lane match, and the /equip command for the build.

 live Enemy Resistances

Sortable table of every enemy × element with red/green coding. Edit per-enemy resistances inline; export an updated enemies.json. Pairs with the resistances field read by actor.gd::_apply_typed_damage.

 design Item Card Mockups

Six tooltip-layout variants rendered side-by-side at three power tiers (Common · Rare · God-tier 0.00001%). Pick the section-divider style you like — Alt-hold expands every card, same as in game.

 live Slot Proposal

Research-backed proposal for new equipment slots — bows, belts, trinkets, orbs, second ring, talismans. Each candidate scored for sprite availability (inv vs positional), DCSS fit, effort. Top recommendations ranked. Click any card for full detail.

Sharing what you build

Each editor has an ⬇ Export button that downloads JSON. To share your design:

  1. Hit Export and save the file locally.
  2. Open a GitHub issue with the JSON contents (or a Discord post if linked from the project).
  3. If it gets accepted, your file lands in data/ next release.

No backend, no auth, no upload — your work stays local until you choose to share it.

Working locally

Clone the repo, then serve tools/ over HTTP so the editors can fetch() data files (browsers block file:// fetch):

git clone https://github.com/deanyo/botter-idle.git
cd botter-idle
python3 -m http.server 8080
# then open http://localhost:8080/tools/
