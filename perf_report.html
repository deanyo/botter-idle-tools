What shipped

Every fix is a real code change with a file:line in the checklist. Tier 0 — playtest blockers or S-effort quick wins on P0/P1. Tier 1 — P1 with M-effort. Tier 2 — P2 playbook regressions + allocation-churn cleanup. Tier 3 — P3 defensive / hygiene. See docs/perf-audit-2026-07-01/PERF_CHECKLIST.md for the full checklist including deferred items.

Tier 0 — playtest blockers & S-effort quick wins
Tier 0Glow-material cache keyed on color only
bot.gd:1088 · paperdoll_renderer.gd:154
Slider drags on FX Tuner used to compile a fresh Web GL program per step because strength and thickness lived in the cache key. Now uniform mutations on the cached material — no recompile.
Tier 0VideoSettings.load_settings cache
video_settings.gd:203
Combat-hot config reads (tunable(), is_effect_enabled()) fired 15-25× per frame and re-parsed user://video_settings.json from disk each time. Now O(1) Dict lookups; cache invalidated only on save_settings().
Tier 0Fog nearest-8 light sort
fog_overlay.gd:126
Web-GL fragment cost caps active lights at 8. Pre-fix picked arbitrary 8; now sorts world lights by distance-to-bot before the slice — near-bot torch always wins.
Tier 0Autosave chain redundant reload
dungeon.gd:2603
Autosave was doing flush_to_save() then SaveState.save_state(SaveState.load_state()) — 3 JSON parses + 2 writes per 60s tick. Dropped the redundant reload.
Tier 0Boss + portal texture prewarm
biome_data.gd:151 · dungeon.gd:2436 · outpost.gd:187
Boss textures lived outside enemy_pool and portal biomes were never in the outpost's unlocked walk. Now prewarmed proactively at Outpost across all Portal.PORTAL_KINDS and defensively at portal entry.
Tier 0Initial spawn stagger
dungeon.gd:5251 · wave_spawner.gd
Pre-fix _spawn_packs synchronously instantiated up to 350 mobs on floor build. Now spawns pack leaders synchronously (floor reads populated immediately) and queues packmates into wave.pending_initial_spawns drained 4/frame from _process.
Tier 1 — P1, M-effort
Tier 1Autosave retry-delay skip
save_state.gd:1024 · run_state.gd:420
_save_wrapper gained retry_delay_ms. Autosave path passes 0 — no OS.delay_msec(1000) stall in _process on transient FS contention. Quit/menu/shop keep the 1s default.
Tier 1Compact save JSON
save_state.gd:1082
Dropped 2-space indent from JSON.stringify on save. Verified live save file shrunk ~14KB → ~5.7KB (60% reduction). Halves stringify+parse cost.
Tier 1Migration walker across all containers
save_state.gd::_walk_all_item_containers
_migrate_to_v8/v10/v13 now route through one helper that hits equipped + inventory + stash + orphaned_items. Pre-fix, several migrations skipped stash and orphaned_items — Steam-Cloud-tanking silent stat loss.
Playbook — CLAUDE.md HTML5 rules amended + new Rule 7
PlaybookRules 1, 3, 4 amended + new Rule 7
CLAUDE.md — HTML5 perf playbook
Rule 1 (cache-key vs uniform mutation), Rule 3 (data-driven prewarm — walk JSON, no hand-lists), Rule 4 (ms-budget stagger), new Rule 7 (config reads are hot-path). Post-amend re-sweep found three new drift sites (portal gateways, ambient decor, spell projectile textures) — all fixed inline.
Tier 2 — P2, playbook regressions + allocation cleanup
Tier 2Glow-cache dedup into UITheme
ui_theme.gd::glow_material_for · bot.gd · paperdoll_renderer.gd
Bot rig and paperdoll each maintained their own _glow_mat_cache — every (color) tuple compiled TWICE on Web GL. Both now delegate to UITheme.glow_material_for. Playbook rule 1 doctrine — one shader program per unique visual.
Tier 2_vis_material static shared instance
map_renderer.gd:334
Was ShaderMaterial.new() per floor build. Grid size and tile size are game-constants; only memory_strength is variable and mutable via uniform. Static shared instance reused across every floor.
Tier 2TileSet cache-key uniqueness
map_renderer.gd:309
Cache key was biome_id + count. Two floors of the same biome with different vault decor rosters could hash-collide and bind the wrong TileSet. Now composite key includes sorted resource-path signature — impossible to collide.
Tier 2AmbientDecor visibility gate
ambient_decor.gd:141,143
Fog-hidden decor kept running noise sampling in _process AND its ember emitter kept spawning particles nothing rendered. Now early-returns + pauses ember_emitter.emitting when hidden.
Tier 2GrindLog print gate
grind_log.gd:20
51 call sites fire per kill/loot/pack. Unconditional print() was dumping them all into the browser console on every player run. Now gated behind _enabled — only fires in grind mode.
Tier 2Data-driven prewarm coverage
biome_data.gd::prewarm_biome
Portal gateways, ambient decor tiles, spell projectiles, item-glow materials — all four now walked from their underlying data structure (Portal.PORTAL_KINDS, AmbientDecor.DECOR_SPECS, SpellData.ARCHETYPES, UITheme.FLAVOR_COLORS). Adding a new entry auto-covers prewarm.
Tier 2HUD debug + minimap 4Hz throttle
dungeon.gd:1259,1268
Debug HUD build ran every frame (~10 string allocs + 3 arrays + Performance.get_monitor per frame). Folded into the same 4Hz throttle as the minimap refresh.
Tier 2_gather_lights reusable Dict pool
dungeon.gd:5013
Was allocating a fresh Array + N Dicts every frame — up to 60 Dict allocs/frame on Forge. Now mutates a reusable pool in place; only trims overshoot.
Tier 2stat_panel hash key drop gold+xp
hud_chrome.gd:3255
StatCalc.compute doesn't derive stats from bot.gold or bot.xp. They were in the hash key so every kill / gold pickup / XP tick forced a full StatCalc recompute + panel re-render. Dropped both from the tuple.
Tier 2ItemsDb consolidation (6 sites)
outpost.gd · main_menu.gd · shop.gd · doctrine_screen.gd
Six _load_items() hand-rolls each parsed the ~875KB items.json from disk on every open. All now delegate to ItemsDb.items() which caches the parse once at startup.
Tier 2VaultLibrary preload at boot
vault_library.gd::preload_all · main.gd
First Deploy paid the ~0.9MB vault-bundle parse cost synchronously. Now warmed at main._ready alongside ItemsDb.preload_all() so outpost → dungeon transition doesn't stall.
Tier 2Outpost prewarm priority
outpost.gd:187
Reordered so state.last_branch warms first — an impatient Deploy click doesn't hit an unwarmed biome.
Tier 2RunState.journal per-floor cap
run_state.gd:377
Journal events grew unbounded — a 30-floor run accumulated ~3000 entries deep-copied on end_run. Now capped at 64 per floor with pop-front on overflow.
Tier 3 — P3, defensive / hygiene
Tier 3Pathfinding Dict-set + skip walkable writes
pathfinding.gd:23
IMPASSABLE_CELLS now a Dictionary for O(1) lookup instead of array in scan. Also skip set_point_solid(false) — AStarGrid2D defaults to non-solid, so we only need to write TRUE for wall cells.
Tier 3Autosave duplicate(true) drop
run_state.gd:417
Autosave was recursively cloning full inventory + equipped every 60s tick (~3000 allocs on stash-heavy save). SaveState.save_state doesn't mutate what it reads, so the deep copy was pure waste. end_run keeps its duplicate since output feeds both persistence AND the run report.
Tier 3_pending_texture_set companion Dict
map_renderer.gd:87
Dedup on _register_texture was tex in _pending_textures — O(N) linear scan. Companion Dict-set gives O(1). On floors with 100 unique tiles the pre-fix cost was O(N²) per tileset build.
Tier 3Minimap signature cheap-hash
hud_chrome.gd:3967
Was grid.hash() — recursively hashed all 80 rows every 0.25s (~7ms). Now uses grid[0].hash() + row count + fog size. Detects both descent (row 0 changes) and fog-reveal growth in <0.1ms.
Tier 3Tooltip position early-return
hud_chrome.gd:596
_process re-clamped tooltip position + re-flowed compare-panels every frame while any tooltip was up. Now early-returns when mouse position + tooltip size both match last frame; skips clamp + fanout on stable hovers.
Tier 3Spell iter_order lazy-duplicate
spell_system.gd:93
Every tick did SPELL_SLOTS.duplicate() even when mana wasn't starved. Non-starved path now iterates the const directly; the duplicate happens only in the rare mana-starvation branch.
Tier 3GridPanel set_grid early-return
grid_panel.gd:58
TargetFrame re-called set_grid every AI tick with usually-identical inputs. Pre-fix, each call queued a stale call_deferred entry even though _apply_grid would dedup. Now short-circuits before the queue push.
Tier 3update_buffs coarse-hash + hoisted Callable
hud_chrome.gd:2665
Fast pre-check statuses.hash() ^ floor(now_ms/250) catches the "stable buffs" common case in <1µs. Sort Callable hoisted to a static function so no per-frame Callable allocation.
Tier 3tick_statuses expired-scratch reuse
actor.gd:2379
Was allocating fresh Array[String] every frame per actor even when no statuses expired — ~150 wasted allocs/frame at 150 alive enemies. Now reuses _expired_scratch member with .clear() at tick start.
Tier 3Dead damaged signal removed
actor.gd:7
signal damaged had zero external connect() sites but was emitted 7+ places per hit including reflect / thorns cascades. Deleted signal + all 6 emit sites — pure hygiene, no behavior change.
